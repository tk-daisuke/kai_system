🌏 タイムゾーン設定
UIトグル（グローバル）

期間バー右の {from} = トグルで、汎用変数（{from} / {to} / {from_date} など）がJSTとUTCのどちらで展開されるかを切り替えます。

JST2026-02-22T00:00:00+09:00
↔
UTC2026-02-21T15:00:00Z
明示的な {from_jst} / {from_utc} はトグルに関係なく常に使用できます。
アクション別 timezone 設定

actions.yaml の timezone フィールドでアクション単位に固定できます。UIトグルより優先されます。

- id: dashboard_scrape name: ダッシュボードデータ取得 type: scraper timezone: utc # ← jst or utc params: url: "https://api.example.com/data ?from={from}&to={to}"
UIバッジ UTC はこのアクションがUTC固定であることを示します。
📝 actions.yaml 使用例
# ─── 週番号でファイル名を生成する例 ─── - id: weekly_report name: 週次レポート取得 type: csv_download timezone: jst params: url: "https://example.com/report?week={week02}&year={from_jst_date}" filename: "report_W{week02}_{from_jst_date}.csv" # ─── UTC の Epoch タイムスタンプをAPIに渡す例 ─── - id: api_export name: API集計エクスポート type: shell_cmd timezone: utc params: cmd: "curl 'https://api.example.com/export ?start={from_epoch}&end={to_epoch}'" # ─── 日勤シフトをJSTファイル名付きで保存する例 ─── - id: shift_data name: シフトデータ取得 type: csv_download params: url: "https://example.com/shift ?from={from_jst_short}&to={to_jst_short}" filename: "shift_{from_jst_date}_{from_jst_time}.csv"
📅 週番号のルール
WEEKNUM 互換（日曜始まり）
ルール 内容
第1週 1月1日が属する週
週の開始 日曜日
互換 Excel WEEKNUM(date, 1)
変数 {week}, {week02}, {from_week}
UIの週番号入力フィールドもこのルールで動作します。
ISO 8601（月曜始まり）
ルール 内容
第1週 その年最初の木曜が属する週
週の開始 月曜日
互換 ISO 8601 / Excel ISOWEEKNUM
変数 {week_iso}, {week_iso02}, {from_week_iso}
年末年始は年番号が前後の年になる場合があります。
